## 9. Blog Posts

### `/blogs/favicli` — favicli
- **Tags:** cli, react, nextjs, tooling, favicon
- **Created:** 2026-02-21
- **Image (live):** https://zainafzal.dev/favicli.png
- **Image (local):** `public/favicli.png` (object-fit: contain)
- **Summary:** One command to set up favicons for React and Next.js projects. Detect → Choose → Generate → Inject.
- **Repo:** https://github.com/sheikhmuhammadzain/favicli
- **Issues:** https://github.com/sheikhmuhammadzain/favicli/issues
- **License:** MIT

**Install:** `npm install -g favicli` or `npx favicli`

**Commands:**
- `favicli` — interactive home menu
- `favicli set [image]` — generate + inject favicons. Flags: `-d, --dir <path>`, `--no-inject`
- `favicli detect` — detect project type
- `favicli remove` — remove generated favicon files

**Generated files (in `public/`):**
`favicon.ico` (16+32+48 combined), `favicon-16x16.png`, `favicon-32x32.png`, `favicon-48x48.png`, `apple-touch-icon.png`, `android-chrome-192x192.png`, `android-chrome-512x512.png`, `site.webmanifest`

**Supported projects:** React + Vite (root `index.html`), Create React App (`public/index.html`), Next.js App Router (`app/layout.tsx` metadata), Next.js Pages Router (`pages/_document.tsx`). Monorepo support: root, direct subfolders, `apps/*`, `packages/*`.

**Notes:** Re-running `favicli set` updates files; injection is replacement-based; recommended source image is square (1:1), ≥512×512.
